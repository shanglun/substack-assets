Loss Functions for Neural Networks

Deep learning · fundamentals

How a network knows it's wrong.

Every loss function answers the same question — how bad was that prediction? — but each one answers it differently, and the choice quietly defines what your model is even trying to learn.

The shape of being wrong

Training a neural network is, at heart, a single repeated act: predict, measure how wrong, adjust. The thing that does the measuring is the loss function. It turns the gap between predictions and truth into one number, and gradient descent uses that number to nudge weights in a slightly less wrong direction. The whole apparatus rests on this single quantity.

What's interesting — and what trips people up — is that the loss function isn't a neutral measurement. It's an editorial choice. A network trained with MSE doesn't just measure the same kind of error as one trained with MAE, weighted more harshly. It's optimizing for a different objective entirely. With MSE, the network learns to predict the conditional mean. With MAE, the conditional median. Same data, same architecture, different learned function. The loss is the objective; change it and you change the meaning of "trained."

Three flavors of regression

MSE  Mean Squared Error

MSE squares the error. An error of 4 contributes 16 to the loss; an error of 2 contributes 4 — four times as much for an error twice as large, not two. This makes MSE obsessed with the outlier. Pull the orange dot far up in the widget above and watch the coral line chase it.

MSE = (1/n) · Σ (yᵢ − ŷᵢ)²

Two practical consequences follow. First, MSE has a gradient that scales smoothly with the error magnitude — large early in training, small near the optimum — which gradient descent loves. Second, it is extremely sensitive to outliers. One mislabelled data point with a huge error can dominate the entire loss surface.

Concrete example

Predicting house prices, true price = $500k.

Predict $510k → error 10, squared = 100
Predict $480k → error 20, squared = 400
Predict $400k → error 100, squared = 10,000

The $100k miss contributes 25× more to the loss than the $20k miss. If that house was a typo, MSE will let it dictate the model.

MAE  Mean Absolute Error

Same idea, without the squaring. A miss of 20 contributes exactly twice as much as a miss of 10 — proportional, not punitive. The result: MAE doesn't care how extreme an outlier is, only that it's wrong. In the widget, the blue MAE line barely moves no matter how far you yank the orange dot, because the median is unmoved by extreme values.

MAE = (1/n) · Σ |yᵢ − ŷᵢ|

The cost of that robustness: the gradient of |x| is ±1 everywhere, so the update signal is the same size whether the model is off by $1 or off by $1,000,000. The model doesn't know it's almost there, and training can oscillate near the optimum rather than settling cleanly into it.

Huber  The diplomatic compromise

Huber loss has a knob — call it δ — that splits the loss into two regimes. For errors smaller than δ, it acts exactly like MSE: smooth, with a gradient that shrinks as the prediction gets closer. For errors larger than δ, it switches to acting like MAE: linear, capped, refusing to be bullied by outliers.

L_δ(e) = ½ e² if |e| ≤ δ,  else δ · (|e| − ½δ)

The δ slider in the widget controls this threshold. Pull δ up high and Huber starts behaving like pure MSE. Pull it down low and it converges on MAE. The sweet spot for noisy regression problems is usually around δ = 1 (after scaling), which is why it's a popular default in deep reinforcement learning where Q-value targets are notoriously noisy.

Classification: the asymmetry of being confident

Classification is fundamentally different from regression in one way: the network's output isn't a number on a real line, it's a probability — or a vector of probabilities summing to one. So the loss function isn't measuring distance, it's measuring something more interesting: how surprised should the model be by the truth?

The standard answer is cross-entropy, which is just the negative log of the probability the model assigned to the correct answer. For binary classification:

BCE = −[ y · log(p̂) + (1 − y) · log(1 − p̂) ]

That formula looks fearsome but only one term is ever active. If the truth is y = 1, the loss is −log(p̂); if y = 0, it's −log(1 − p̂). Either way: the negative log of the probability assigned to whatever turned out to be true.

Switch the widget above to the Classification tab and you can see what this shape does. Drag the predicted-probability marker. When the model is confidently correct (probability ≈ 1 on the right answer), the loss is tiny. When uncertain, it's moderate. When confidently wrong, it shoots up — not linearly, but logarithmically. A model that says "99% sure" and is wrong gets crushed; a model that says "60% sure" and is wrong gets a gentle nudge.

Why the asymmetry matters

Cross-entropy doesn't just want the model to be right — it wants the model's confidence to mean something. A 90% prediction should be right roughly 90% of the time. This property is called calibration, and it's what makes cross-entropy the right loss for any task where you need probabilities you can act on (medical triage, fraud detection, search ranking).

For multi-class problems, categorical cross-entropy is the same idea generalised to many classes via a softmax output. The loss collapses to −log(p̂ of the correct class) — same intuition, just with more options to spread probability across. This is the loss running underneath ImageNet classifiers, Fashion MNIST tutorials, and the next-token prediction objective inside every large language model.

Beyond the basics

Most production systems use one of these four — MSE/Huber for regression, binary or categorical cross-entropy for classification. But there's a whole zoo of specialised losses worth knowing exist:

Focal loss multiplies cross-entropy by (1 − p̂)γ, which down-weights examples the model already gets right and concentrates gradients on the hard ones. Indispensable for severe class imbalance — fraud detection, object detection (where 99% of pixels are background), rare disease classification.

KL divergence measures distance between two probability distributions, used whenever your target is a distribution (knowledge distillation, variational autoencoders, label smoothing). Categorical cross-entropy turns out to be KL divergence with a one-hot target.

Contrastive and triplet losses train embeddings — pushing similar inputs close together in some learned space and dissimilar inputs apart. The machinery behind face recognition (FaceNet), image retrieval, and most semantic search systems.

Dice loss directly optimises the overlap between predicted and ground-truth masks in semantic segmentation. Pixel-wise cross-entropy gets swamped by easy background pixels; Dice doesn't.

The one rule, and the meta-rule

The single most common silent bug in neural network code is mismatching the output activation and the loss function. They have to be paired correctly: sigmoid + binary cross-entropy, softmax + categorical cross-entropy, linear + MSE/MAE/Huber. A softmax output trained with MSE is technically legal code that will train slowly and learn the wrong thing. (Modern frameworks like PyTorch and TensorFlow paper over this by fusing the activation into the loss — CrossEntropyLoss expects raw logits, not softmaxed probabilities. The fused version is more numerically stable.)

The meta-rule: the loss function is a hyperparameter, not a constant. Treat it like learning rate or architecture choice — start with the default for your task, but be willing to experiment when the default isn't doing what you need. The choice of loss is, ultimately, a choice about what kinds of errors you're willing to live with and which ones you want to obliterate.
