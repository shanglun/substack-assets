I.

How a CNN sees.

The problem with dense networks

A dense network sees pixels as strangers.

Two pixels next to each other have no more relationship — as far as the network is concerned — than two pixels on opposite corners.
The fix

Two architectural commitments.

  • Locality. Neurons only look at small neighborhoods.
  • Translation invariance. The same small filter gets reused everywhere.

Filters as questions

Each filter has an ideal patch.

A filter fires strongest when the image patch under it looks like the filter itself. Dot product, maximized when aligned.
Stride

Bigger stride, smaller output.

The filter jumps multiple pixels per step. Cheaper to compute, coarser resolution. Stride 2 roughly halves the feature map.
Padding

A ring of zeros to keep size.

Without padding, every conv layer shrinks the image — you run out of pixels before you can build a deep network. "Same" padding fixes that.
Pooling

Compress the feature map. Keep the loudest signal.

Hierarchy of features

Each layer builds on the last.

Layer 01
Edges & gradients
Bars and corners at various orientations. Almost identical to the edge detectors we played with.
Layer 02–03
Textures & patterns
Combinations of edges. Stripes, dots, simple curves, repeating motifs.
Layer 04–05
Parts
Eyes, wheels, leaves, the tip of a rice grain. Semantically meaningful pieces of objects.
Layer 06+
Whole objects
Faces, cars, particular rice varieties. The hierarchy is not designed — it's what the math settles into.
Part Two
II.

What lights up.

The core insight

A filter is a dot product wearing a costume.

Maximized when the patch and the filter point in the same direction. Every filter has an "ideal image" — the precise arrangement of pixels that would produce its maximum response.

The takeaways

Six pictures, six lessons.

  • Orientation selectivity. Each filter fires on edges in its direction, ignores the rest.
  • Filters ignore boring regions. Smooth gradients barely activate anything.
  • Multiple filters together describe an object: where its edges are, where its center sits.
  • Edge + edge nearby becomes a corner. That's what layer 2 will start looking for.
Hyperparameter 01 · Filter size

Use 3×3. Everywhere.

Two stacked 3×3 layers see the same 5×5 receptive field but use fewer weights and add a non-linearity. More expressive, cheaper, easier to optimize.
Hyperparameter 02 · Filter count

Double the filters each time spatial size halves.

16 → 32 → 64 → 128. As spatial info shrinks, you can afford more channels. The "pyramid" pattern.
Hyperparameters 03 & 04 · Downsampling and padding

Stride 1 + 2×2 pool. Same padding.

Let pooling do the explicit downsampling. Keep conv layers shape-preserving so they compose freely. The modern alternative is stride-2 convolution — equally valid, learnable.

Part Three
III.

How a filter learns.

What backprop is actually doing

Would the loss go up or down if I nudged this one weight?

Answer that for every weight, every step. The gradient points downhill. Take a small step. The network gets slightly better. Repeat ten thousand times.
The conv layer in reverse

One signal arrives. Two signals leave.

Output 1 of 2

The filter gradient.

∂L/∂w[a,b] = Σi,j (∂L/∂y[i,j]) × x[i+a, j+b] Each filter weight gets used at every output position. Sum its contribution to error across all those positions. That sum, scaled by the learning rate, is how much we nudge the weight.
Output 2 of 2

The input gradient.

∂L/∂x[m,n] = Σi,j (∂L/∂y[i,j]) × w[m−i, n−j] Each input pixel was used by multiple output positions. Collect their gradient contributions, weighted by the filter weight that touched it there. Hand the result down to the previous layer.
The elegant fact

Backprop through a conv layer is a convolution.

Forward: conv(x, w). Backward: conv(x, ∂L/∂y) for the weight gradient. Conv(∂L/∂y, flipped w) for the input gradient. Same hardware. Same kernels. Same operation, just running in different directions.

The hidden advantage

One filter, thousands of gradient updates per image.

In a dense layer, each weight is used once per example — one gradient contribution. In a conv layer, each weight is used at every spatial position. Two orders of magnitude more learning signal per parameter.
